請閱讀 docs/examples/standterm-external-agent-skill/SKILL.md，增加 `standterm-external-agent` local skill；若使用者要求操作 StandTerm，從 StandTerm startup banner 或該 skill 取得 active Python、`scripts/agent_cli.py`、`scripts/agent_jsonl.py`、`scripts/agent_repl.py`、`scripts/agent_type.py`、`standterm_external_agent_handoff.json` 的絕對路徑，不要猜 port、URL 或 token；舊 `webssh_agent_*.py` 與 `webssh_external_agent_handoff.json` 只當 compatibility alias。
使用時先確認 handoff 是瀏覽器 Agent UI 新 mint 的 local secret，不印出 token；先跑 `hello` 並只依 typed JSON fields 決策，連續機器操作優先用 JSONL client，單步優先 `send-wait`/`screen --tail-lines`/`screen --region`；CLI `--text` 不解讀 `\r`/`\n` escape，bash 需用 `$'...\r'`，Windows 優先用 `--stdin` 或 JSONL；ANSI redraw 太吵才加 `tail --strip-ansi` 或 `send-wait --strip-ansi`，但 strip 後可能看不到游標/反白，選單操作前先確認目前 view state；若 unauthorized 且舊 handoff 是非 loopback host，先用同 port 的 loopback URL 重試，expired/revoked 才請使用者重新 mint token。
